How shall I start.. Let's see, 4 years ago I was trying to create a minimal Twitter bot, a very small tiny thing it was intended to response to some kind of tweets with a message and a smiley face, far form thinking of how to implement this naive complexity programmatically, I was working on writing a formal and appropriate 'epistle' to convince Twitter's development support that I really deserve having a developer account with API key. Here I knew that "Twitter developer" will not be a very nice experience whatsoever.

And days go by, and I needed to create this project.. this poor project, I wanted to have a Twitter account for a Facebook page that I used to post on (nothing serious, just old paintings and poetry) yeah I know that there is already bunch of applications already implement that, but I wanted something simple and easy, just authentication from here, put it here, connect to the APIs… and that's it. I shouldn't see a 'pricing' page in such a process.

I've successfully passed the first hell, Twitter's development support, I convinced him/her after 4 email texts to give me access do you remember that blog about Oracle? It's quite the same thing, nonetheless nobody could sign up on Twitter if Oracle was on the lead.

The biggest pain was to understand the API docs. I don't know how to explain it, it just didn't make any sense, it felt like I'm trying to solve a puzzle.
